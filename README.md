# 综测分部分条目自动检查程序

ScoreCheck是作者在学院综测分的手工检查中发现部分项目使用的原始数据存在结构化特征，如果使用传统的ctrl+F逐一查找的话会浪费大量的时间精力；且最终的审查签字和总分核算汇总需要反复打开不同excel表格，同样存在优化空间。因此在之前撰写的小脚本的基础上整合为一个综合性程序，用以自动化某些工序。

## ❗重要提示❗

 **本程序仅用于自动化综测分检查的部分工序，不对原始文件数据和非本程序涉及的检查质量做出保证。使用签字功能时请确保检查组成员均已认真完成工作，并且在程序使用结束以后及时抽检确认。由于本程序功能之外的原因造成的不利影响，本程序及开发者不承担相关责任。**

## 使用方法

### 使用前准备

- 安装python3和以下依赖库（程序内置依赖库的检查和自动安装，采用清华镜像源，也可以手动提前安装）

```
xlwings
openpyxl
pandas
```

- data目录中需要包含以下文件（系统导出后重命名，不要做出其他的改动，具体字段模板可以参考【参考文件】文件夹内的data_template.xlsx文件）

```
PE.xlsx  体育成绩总表
dom.xlsx  寝室卫生情况总表
members.xlsx  班干部考核表
names 班级花名册
```

* config.json文件用于存储使用功能4和功能5时使用的签字人员配置，格式如下
  【注意：JSON文件本身不支持注释，此处通过C++格式注释解释对应字段，但是正式文件中写入注释会报错。使用时可以根据 **参考文件** 文件夹中的 **config_example.json** 进行更改】

  ```json
    {
        "signature_config": {
            "current_class": {                                  //该大括号内表示本班签字的信息
                "reviewers": ["李四", "王五", "赵六"],          //一校人员名单
                "secondary_reviewer": "张三",                   //二校人员
                "header_position": "f3"                         //表示添加的表格起始位置，即“一校”文本所在的位置
            },
            "other_class": {                                    //该大括号内表示其他班审核签字的信息
                "reviewers": ["李四", "王五", "赵六", "张三"],   //审核签字人员名单
                "name_cell": "h4",                              //签字位置
                "score_cell": "i4"                              //总分填写位置
            }
        }
    }
  ```

### 选择需要使用的功能

双击scorecheck.py主程序或通过IDE打开后运行即启动系统。

系统启动后会进行相关目录的创建，并提示将所需的文件放入data文件夹。完成后程序会自动配置数据库，并提示将待检查的文件放入test_data文件夹中。当一切准备完成的时候，终端会显示菜单，一共有五个选项，两大种功能，接下来将详细介绍。

菜单示例如下：

```
1 - 检查体育成绩"
2 - 检查寝室成绩"
3 - 检查班干部工作"
4 - 本班审核和计算签字"
5 - 其他班审核和计算签字"
```

1. #### 体育成绩、寝室成绩和班干部考核

   选项1、2、3分别为体育、寝室和班干部考核的选项，可以单选也可以多选，将想要的功能直接输入即可，数字之间不需要有分隔符号，随后程序就会自动根据之前存储在数据库里的数据，从待检查的表格中获取学号、姓名和自己填写的分数，并和数据库中获取的成绩经过规则匹配之后得到的分数进行对比，如果有误则记录在输出的表格中。同时支持登记花名册，以避免漏记。

   需要特殊说明的是，这三项所采用的计分规则是HDU会计学院现行规则，如果需要改动需要在程序内进行改动。班干部考核采用的是二课系统导出的班干部评价表，但是学年纪实测评表中读取的项目为“社会工作”，因此可能会导致学院和学校工作无法获取数据而出现匹配错误的情况，目前需要人工核实。
2. #### 本班签字、其他班审核签字和总分汇总

   选项4为本班签字功能，选项5为审核其他班的签字功能。两者均附带计算总分和输出总分列表功能。在选择4或5之后，会自动加载config.json文件以读取签字人信息和填写位置，并打印在终端供用户确认。

   本班签字功能名字写在“一校”和“二校”位置，其他班签字功能名字写在“三校”位置，总分不受影响。

   总分计算会从第5行开始，并自动识别“其他”项所在的行作为最后一行。考虑到总分可能填写在D或E列，因此程序中默认求和列为D，但是用户可以手动选择E列作为求和列。

选项123均可单独使用，也可配合使用，也可和45其一配合使用。选项45为互斥选项，不可同时选取。选取完成之后程序即可自动运行直至最终输出结果。程序运行时请不要操作相关Excel文件，以免出现错误。

### 结果的输出

选项123不会对待查文件本身产生操作，程序运行完成以后会生成一个“报错汇总.xlsx”文件，里面存储了可能存在问题的人员姓名，可能存在问题项，表格中分数和数据库比对分数。需要指出的是，数据库比对错误可能是由于数据库出现问题（例如上述工作匹配问题），需要人工进行二次核实，请勿盲从。

选项45在程序运行完成以后，会生成一个“个人综测分.xlsx”文件，里面存储了经过检查的人员的学号、姓名和综测分总额。另外，这两项操作会在待查文件中直接编辑输入对应的表格、姓名和分数，因此如果需要导出处理完成的文件直接将test_data文件夹打包即可。

### 下一步开发计划

下一步开发计划将分为两个部分进行

1. #### 对现有程序的完善

   现有程序在之前分部分的小脚本上整理而来，在部分内容例如社会工作上的自动比对有一定不足；且当前的评分规则直接在程序中编入，未能迁移到配置文件内，如果需要修改略有不变；同时部分用户可能会不习惯终端操作。后续将探讨如何在现有框架下优化程序功能，提升用户体验。
2. #### 探讨更多工序的自动化

   当前程序是基于已经填好的表格并对其部分内容进行自动化处理，如果想要实现更多非标工序尽可能自动化，减轻综测分审核人员的压力，可能需要对整个综测评分的工作流进行自动化重构。
